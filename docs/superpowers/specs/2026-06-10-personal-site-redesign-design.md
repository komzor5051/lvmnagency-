# Редизайн lvmn-site: персональный сайт с продаваемыми продуктами

Дата: 2026-06-10
Статус: согласовано с Владом (брейншторминг с визуальным компаньоном)

## Цель

Превратить сайт из «личной студии AI-консалтинга» в персональный сайт Влада Лямина,
который продаёт: консультации и цифровые продукты покупаются онлайн через lava.top,
крупные услуги (внедрение, advisory) идут через заявку.

Диагноз текущей версии (editorial, бежевая): скучная, старомодная, не продаёт.

## Фазирование

- **Фаза 1 (этот проект)**: редизайн всех страниц, продукты, анимации A + B + D.

  за заголовком hero слева, реагирует на мышь. Фото справа остаётся главным визуалом.

## Главная — порядок секций

1. **Nav**: имя слева, ссылки (Продукты, Кейсы, Обо мне, Блог), чёрная кнопка «Консультация».
2. **Hero**: kicker моноширинным (AI-ИНЖЕНЕР · НОВОСИБИРСК · ОТКРЫТ К ПРОЕКТАМ);
   заголовок «Внедряю AI-системы, которые окупаются, а не презентуются» с рисованной
   оранжевой SVG-линией под «окупаются» (прорисовывается при загрузке); подзаголовок
   от первого лица; CTA «Консультация — 10 000 ₽» (чёрная) + «Смотреть продукты ↓»
   (оранжевое подчёркивание). Справа фото 4:5 с 3D-tilt за курсором и парящими
   плашками на разной глубине: «40+ внедрений», «сейчас: 1 слот», «n8n · Claude · Supabase».
   Заголовок собирается по строкам при загрузке.
3. **Полоса цифр**: 40+ внедрений, 50+ обученных, ×1.7 конверсия, −38% времени.
   Count-up при появлении (паттерн из gsap-demo).
4. **Витрина продуктов** (заголовок «Что можно купить прямо сейчас», номер секции
   оранжевым моноширинным): grid 2×2 из `lib/products.ts`. Карточки каскадом при
   скролле; hover — подъём + оранжевая линия сверху + 3D-tilt с бликом.
5. **Тёмный блок «для бизнеса»** (единственная тёмная секция, скруглённый чёрный
   блок с декоративными окружностями): внедрение от 50 000 ₽, обучение, advisory.
   CTA «Рассказать о задаче» → Telegram/форма. Наезжает на витрину с parallax (scroll-сцена).
6. **Кейсы**: строки таблицей — крупная оранжевая цифра, описание, тег моноширинным.
   Появление построчно, цифры одометром.
7. **Тизер «обо мне»**: живое фото + «Я не агентство. Я инженер, который отвечает
   лично» + ссылка на /about с оранжевым подчёркиванием.
8. **Финальный CTA**: «Начнём с часа?» + кнопка консультации + «или напишите в Telegram @lyaminvl».

## Продукты — модель данных

Один типизированный файл `lib/products.ts`, без БД:

```ts
type Product = {
  id: string;
  type: "consultation" | "digital" | "service" | "coming-soon";
  title: string;
  tagline: string;          // короткое описание для карточки
  description: string[];     // абзацы для /products
  price: number | null;      // null для coming-soon
  priceLabel: string;        // «10 000 ₽», «от 50 000 ₽»
  meta: string;              // «1:1 · 60 минут · онлайн»
  buy:
    | { kind: "lava"; url: string }     // чекаут lava.top
    | { kind: "form"; url: string }     // /audit или форма заявки
    | { kind: "waitlist" };             // сбор email (курс)
  faq?: { q: string; a: string }[];
};
```

Стартовый состав: консультация (lava), гайд (lava), AI-аудит (form → /audit),
мини-курс (waitlist). Добавить продукт = добавить объект.

Waitlist курса: форма email → существующий механизм лидов в Notion CRM
(тот же API-роут, что у аудит-формы, с пометкой source: course-waitlist).

## Дизайн-система

- Токены в одном модуле (`lib/design-tokens.ts` или CSS-переменные в globals):
  bg #fafafa, fg #111, muted #555, line #e8e8e8, accent #ff4d00.
- Правило акцента: оранжевый не заливает кнопки и фоны — только линии,
  подчёркивания, маркеры, мелкие индикаторы. Кнопки: чёрная заливка / чёрная обводка.
- Типографика: гротеск Inter-класса с характером (выбрать при имплементации:
  Inter Tight / Manrope / Unbounded для заголовков — решение за исполнителем,
  критерий: современно, не дефолтный Inter), моноширинный для kicker/меток.
  Никаких serif — старый editorial-стиль уходит целиком.
- Заголовки: крупные, жирные (700), letter-spacing −0.03…−0.04em.

## Motion-система

Всё на GSAP + ScrollTrigger через `@gsap/react` (`useGSAP` с автоочисткой —
паттерны уже отработаны в `app/gsap-demo/page.tsx`):

- Появление секций: батчевый stagger (карточки каскадом).
- Count-up цифр: tween прокси-объекта, once: true.
- SVG-линия под «окупаются»: stroke-dashoffset прорисовка при загрузке.
- Hero-заголовок: сборка по строкам (clip + y-shift).
- Scroll-сцены: тёмный блок с parallax-наездом; цифры кейсов одометром.
- 3D-tilt (фото hero, карточки продуктов): собственный хук на mousemove
  (rotateX/rotateY + glare-блик), без сторонних библиотек.
- Доступность: `prefers-reduced-motion` отключает всё движение;
  на тач-устройствах tilt выключен, анимации появления остаются.

## Ошибки и граничные случаи

- Ссылки lava.top открываются в новой вкладке; если продукт без url (не настроен) —
  карточка рендерится с кнопкой-заглушкой «скоро» (билд не падает).
- Waitlist-форма: невалидный email — инлайн-ошибка; сбой API — сообщение
  «напишите в Telegram» с ссылкой (деградация в ручной канал).
- Изображения: фото через next/image с приоритетом в hero.

## Тестирование

- `npm run build` и `npm run lint` зелёные.
- Ручная проверка: все CTA ведут куда надо (lava.top, /audit, Telegram),
  redirect /employees → /, блог открывается в новом стиле.
- Lighthouse на главной: performance ≥ 85 mobile (анимации не должны уронить).
- Проверка `prefers-reduced-motion` и мобильной вёрстки (tilt выключен).

## Вне скоупа (фаза 2+)

- WebGL-граф в hero (R3F).
- Онлайн-покупка аудита, запуск курса.
- Интеграция расписания/брони для консультации (пока lava.top + ручная координация).
